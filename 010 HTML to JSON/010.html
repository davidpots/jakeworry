This all relates to the Jake Worry exercises 008, 009, and 010. These concern a general goal I have to store data in JSON format and then retrieve it for use in HTML. I'm curious about the structure of the JSON -- is it (i.e., the JSON array) multiple levels deep or a single level? How does this effect the commands I use to fetch the JSON into a jQuery object? How does it effect my use of the HTML-to-JSON plugin I use in exercise 010?


    
Example #1


    
Suppose you have a JSON array that has all the objects of note at the highest level (and stuff isn't nested out of the gate):




    [
        { "title":"Pinball Wizard" },
        { "title":"Rocket Man"     },
        { "title":"Space Oddity"   }
    ]



Then, to pull this JSON into use for jQuery you'd do this:



    $.getJSON('data.json', function(json) {
        var size = 0;
        $.each(json, function(i) {
          size++;
          console.log(size);
        });
    });



And like this:



    var randomSong = json[randEntry-1].title;



Example 2



Suppose everything you want in JSON is inside the `songs` object (a level deep), like this:



    { "songs":
        [
            { "title":"Pinball Wizard" },
            { "title":"Rocket Man"     },
            { "title":"Space Oddity"   }
        ]
    }



You'd fetch it like this:



    $.getJSON('data.json', function(json) {
        var size = 0;
        $.each(json.songs, function(i) {
          size++;
        });
    });



and then you'd refer to elements within the array like this:



    var randomSong = json.songs[randEntry-1].title;
